(inner) Import Syntax

To use Imports, put your imports at the top of the file like in the example. "<import>" and "</import>" are used to mark said imports. This is critical, as otherwise, imports wont work. They need to be on separate lines. Note that some editors will autofill incorrect paths, using the local file system to determine relative paths. This obviously needs to be changed, the correct path is /js/, since everything on the clients end is relative to the /public/ directory. Imports are only available for system applications, by design and definition. Non-system applications must not have the same level of access. Conversely, system apps often do need the access to system functionality.
